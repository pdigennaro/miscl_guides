---

## ‚öôÔ∏è What Are Side Effects?

In Jetpack Compose, **UI = f(state)** ‚Äî composables are *pure functions* of state.
But real apps need to do things *outside Compose* (like logging, fetching data, showing toasts, etc.).

Those are **side effects** ‚Äî operations that affect the outside world or need to run beyond the scope of recomposition.

Compose provides **controlled effect APIs** to handle such cases safely and predictably.

---

## üß© The Seven Major Side Effect APIs

| API                          | Purpose                                                   | Coroutine? | Cleanup?                  | Triggered When                                | Common Use                                                   |
| ---------------------------- | --------------------------------------------------------- | ---------- | ------------------------- | --------------------------------------------- | ------------------------------------------------------------ |
| **`LaunchedEffect`**         | Run suspend functions in a composition-aware coroutine    | ‚úÖ          | Auto cancel on key change | First composition or key change               | Fetch data, navigate, delay, animate                         |
| **`rememberCoroutineScope`** | Get a scope tied to composition lifecycle                 | ‚úÖ (manual) | ‚ùå                         | On demand                                     | Launch coroutines on user events                             |
| **`SideEffect`**             | Run synchronous code after every recomposition            | ‚ùå          | ‚ùå                         | After every recomposition                     | Logging, updating external UI state                          |
| **`DisposableEffect`**       | Run setup/cleanup logic when entering/leaving composition | ‚ùå          | ‚úÖ manual                  | On first composition, key change, or disposal | Register/unregister listeners                                |
| **`produceState`**           | Bridge async data sources into Compose State              | ‚úÖ          | Auto cancel               | On first composition or key change            | Collect Flow, load data                                      |
| **`derivedStateOf`**         | Efficiently compute derived values from other State       | ‚ùå          | ‚ùå                         | When dependency changes                       | Derived calculations (e.g., total price)                     |
| **`snapshotFlow`**           | Convert Compose `State` into Flow                         | ‚úÖ          | Auto cancel               | When state snapshot changes                   | Observe state in ViewModel / Flow                            |
| **`rememberUpdatedState`**   | Keep latest value accessible inside stable effects        | ‚ùå          | ‚ùå                         | On recomposition                              | Stable callbacks or latest state inside long-running effects |

## üîπ 1. `LaunchedEffect`

Runs a coroutine **automatically** when a composable first enters composition or when its key(s) change.

```kotlin
@Composable
fun WelcomeScreen(userId: String) {
    LaunchedEffect(userId) {
        val user = fetchUser(userId) // suspend function
        println("Fetched user: $user")
    }

    Text("Welcome, $userId!")
}
```

‚úÖ Cancels old coroutine if `userId` changes.
‚úÖ Perfect for one-time or key-dependent tasks.

---

## üîπ 2. `rememberCoroutineScope`

Use for **imperative event-driven coroutines** (e.g., button clicks).

```kotlin
@Composable
fun DownloadButton() {
    val scope = rememberCoroutineScope()

    Button(onClick = {
        scope.launch { startDownload() }
    }) {
        Text("Download")
    }
}
```

‚úÖ The scope is tied to the composable‚Äôs lifecycle.
‚ùå Doesn‚Äôt automatically restart or cancel unless the composable leaves composition.

---

## üîπ 3. `SideEffect`

Runs **after every recomposition** (once it succeeds).
Used for updating external states or calling non-suspend functions.

```kotlin
@Composable
fun LogState(value: String) {
    SideEffect {
        println("New value: $value")
    }
    Text(value)
}
```

‚ö†Ô∏è No coroutine ‚Äî it‚Äôs synchronous.
Useful for external synchronization or logging.

---

## üîπ 4. `DisposableEffect`

Use when you need **setup and teardown logic** (register listeners, observers, etc.).

```kotlin
@Composable
fun LocationTracker(locationManager: LocationManager) {
    DisposableEffect(Unit) {
        val listener = LocationListener { location ->
            println("Location: $location")
        }
        locationManager.addListener(listener)

        onDispose {
            locationManager.removeListener(listener)
        }
    }
}
```

‚úÖ Automatically runs cleanup when leaving composition or when keys change.

---

## üîπ 5. `produceState`

Creates and exposes a **State<T>** from async or external sources.

```kotlin
@Composable
fun UserName(userId: String): State<String> {
    return produceState(initialValue = "Loading...", userId) {
        value = fetchUserName(userId)
    }
}

@Composable
fun Profile(userId: String) {
    val name by UserName(userId)
    Text("Hello $name")
}
```

‚úÖ Automatically launches/cancels coroutine based on composition.
‚úÖ Used to **bridge non-Compose async data** into Compose world.

---

## üîπ 6. `derivedStateOf`

Used to derive **computed state** from other state values efficiently.

```kotlin
@Composable
fun CartSummary(items: List<CartItem>) {
    val total by remember(items) {
        derivedStateOf { items.sumOf { it.price } }
    }

    Text("Total: $${total}")
}
```

‚úÖ Prevents unnecessary recompositions by recalculating only when input changes.

---

## üîπ 7. `snapshotFlow`

Turns Compose state changes into a **Flow** stream.

```kotlin
@Composable
fun ObserveScroll(scrollState: ScrollState) {
    LaunchedEffect(Unit) {
        snapshotFlow { scrollState.value }
            .collect { println("Scroll: $it") }
    }
}
```

‚úÖ Great for observing Compose state from coroutines or ViewModels.

---

## üîπ 8. `rememberUpdatedState`

Keeps the **latest value or callback** accessible inside long-running effects without restarting them.

### Without it (‚ùå may call stale callback):

```kotlin
LaunchedEffect(Unit) {
    delay(5000)
    onTimeout() // might be old version
}
```

### With it (‚úÖ always fresh):

```kotlin
val latestOnTimeout by rememberUpdatedState(onTimeout)

LaunchedEffect(Unit) {
    delay(5000)
    latestOnTimeout() // always current
}
```

‚úÖ Keeps effect stable
‚úÖ Reads latest values safely
‚úÖ Common for callbacks, state references, or config values

## ‚úÖ Best Practices

1. **Use `LaunchedEffect`** for automatic coroutines tied to lifecycle
2. **Use `rememberCoroutineScope`** for user-triggered actions
3. **Never run suspend functions directly in composables**
4. **Always cancel listeners in `DisposableEffect.onDispose`**
5. **Wrap changing callbacks in `rememberUpdatedState`**
6. **Use `derivedStateOf` to avoid expensive recomputation**
7. **Use `snapshotFlow` when bridging Compose ‚Üí Flow**
8. **Don‚Äôt misuse keys** ‚Äî keys control when effects restart!

## üßæ Example Putting It All Together

```kotlin
@Composable
fun UserProfileScreen(
    userId: String,
    viewModel: UserViewModel,
    onTimeout: () -> Unit
) {
    val latestOnTimeout by rememberUpdatedState(onTimeout)

    // Fetch user when userId changes
    LaunchedEffect(userId) {
        viewModel.loadUser(userId)
    }

    val user by viewModel.user.collectAsState()

    // Derived premium state
    val isPremium by remember(user) {
        derivedStateOf { user.subscription == "premium" }
    }

    // Log recompositions
    SideEffect {
        println("UserProfile recomposed for ${user.name}")
    }

    // Handle cleanup when leaving screen
    DisposableEffect(Unit) {
        registerAnalyticsListener(userId)
        onDispose { unregisterAnalyticsListener(userId) }
    }

    // Timeout after 5 seconds
    LaunchedEffect(Unit) {
        delay(5000)
        latestOnTimeout()
    }

    Text(if (isPremium) "Welcome Premium User!" else "Welcome Free User!")
}
```
